# Contributing to Quote Genie

Thank you for your interest in contributing to Quote Genie! We welcome contributions from the open-source community to make our predictive pricing engine even better.

## ü§ù How to Contribute

### 1. Reporting Issues
*   Check the issue tracker to see if the bug or feature request has already been reported.
*   If not, open a new issue with a clear title and description.
*   Include steps to reproduce the issue, expected behavior, and actual behavior.

### 2. Suggesting Enhancements
*   Open a new issue describing your proposed enhancement.
*   Explain the motivation behind the change and how it benefits the project.
*   Provide any relevant mockups or diagrams.

### 3. Pull Requests
*   Fork the repository and clone it to your local machine.
*   Create a new branch for your feature or bug fix:
    ```bash
    git checkout -b feature/my-new-feature
    ```
*   Implement your changes, adhering to the coding standards.
*   Write or update tests to cover your changes.
*   Ensure all tests pass before submitting your pull request.
*   Commit your changes with descriptive commit messages.
*   Push your branch to your fork:
    ```bash
    git push origin feature/my-new-feature
    ```
*   Open a pull request against the `main` branch of the original repository.

## üíª Coding Standards

### Python (Backend & ML)
*   follow **PEP 8** style guide.
*   Use type hints for function arguments and return values.
*   Write docstrings for all functions and classes.
*   Keep functions small and focused.

### JavaScript/React (Frontend)
*   Use functional components with Hooks.
*   Follow clean code principles (DRY, meaningful variable names).
*   Use modern ES6+ syntax.

## üß™ Testing

*   **Backend**: Run unit tests using `pytest` (setup coming soon).
*   **Frontend**: Run tests using `npm test` (setup coming soon).
*   **Manual Testing**: Verify core flows (Quote Generation, API connectivity) before submitting.
